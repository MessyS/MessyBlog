【预备知识】

在客机与服务器模式下，文件服务器（file server）是一台对中央存储和数据文件管理负责的计算机，这样在同一网络中的其他计算机就可以访问这些文件。文件服务器允许用户在网络上共享信息，而不用通过软磁盘或一些其它外部存储设备来物理地移动文件。任何计算机都能被设置为主机，并作为文件服务器（file server）运行。最简单的形式是，文件服务器可以是一台普通的个人计算机，它处理文件要求并在网络中发送它们。在更复杂的网络中，文件服务器也可以是一台专门的网络附加存储（NAS）设备，它也可以作为其他计算机的远程硬盘驱动器来运行，并允许网络中的人像在他们自己的硬盘中一样在服务器中存储文件。文件服务器具有分时系统文件管理的全部功能，提供网络用户访问文件、目录的并发控制和安全保密措施的局域网（LAN）服务器。

【实验步骤】

网络拓扑：server2008-basic--win7

windows server 2008
用户：administrator
密码：Admin123

win7
用户：test
密码：123456

第一步：点击启动选项，启动实验虚拟机。

第二步：打开windows server 2008虚拟机，进入到系统桌面环境。

第三步：在任务栏中点击“开始”，找到“管理工具”后，点击“共享和存储管理”进入共享和存储管理界面进行操作。

第四步：在共享和存储管理界面中，点击“设置共享”，进入设置共享文件向导

第五步：点击浏览选择要共享文件夹的位置。（例如d:\smb\test1）

第六步：点击“编辑权限”，给共享文件test1添加用户并赋予用户权限。

（只保留test2，Administrator，Administrators）

第七步：选择SMB协议，填写文件共享名（例如test1）

第八步：点击“高级”，进行用户限制和基于访问限制的枚举。

第九步：选择“用户和组具有自定义共享权限”后。点击“权限”进行权限上的分配部署。给everyone用户，赋予改写和读取的权限。

第十步：确认信息，点击“创建”即可。

第十一步：仿照步骤五到十，完成对test2文件夹共享的操作。

 （只保留test2，Administrator，Administrators）

第十二步：测试

在windows server2008中，按键盘的windows键和R键，开启运行窗口，在运行窗口中输入cmd，打开命令指示符。

在命令指示符中输入“ipconfig”，看查本机IP

打开另一台虚拟机windows7，在键盘上按windows键和R键，打开运行窗口，输入cmd打开命令指示符。

输入“ipconfig”看查本机ip，得知server和pc属于同网段网络。

在windows7上，打开计算机，进入计算机界面，在路径栏里输入server的地址。

弹出登录的窗口，需要输入在server上建立的用户和密码（test和Admin123）

登录后点击test1文件，发现可以进入。

点击test2文件，无访问权限。
